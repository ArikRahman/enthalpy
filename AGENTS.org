* Project Overview
Enthalpy is a ClojureScript port and rework of the ColemakCamp typing trainer
(https://github.com/ColemakCamp/colemakcamp.github.io), reoriented around the
Enthium v14 keyboard layout. The primary feature is an interactive typing trainer
with progressive level unlocking, a live cheatsheet keyboard, and WPM/accuracy
scoring. Secondary features are supplementary documentation pages (rationale,
stats, comparisons, learning guide) reachable via hash routes.

The original ColemakCamp source (JavaScript) lives in the repository root and
serves as the reference implementation for UI behaviour and word-list logic.
The ClojureScript rewrite lives in ``src/`` and compiles to ``docs/``.

** What ColemakCamp provides (reference only — do not ship as-is)
- Typing trainer loop: prompt words → capture input → score hits/errors → next
- Progressive level system: each level unlocks more letters (lvl1–lvl8)
- Cheatsheet keyboard: visual key map shown alongside the typing area
- Settings: capital letters, punctuation, backspace correction, time/word limits,
  sound on keystroke/error
- Layout selector: Colemak, Colemak-DH, Tarmak, QWERTY, custom, etc.
- Word list: filtered by the letters available at the current level

** What Enthalpy changes
- Single layout: Enthium v14 only. The layout selector is removed.
- Level letter sets: redefined in Enthium frequency order (highest-freq letters
  first), not Colemak order.
- Branding, copy, and colours: Enthium/Enthalpy identity.
- Stack: ClojureScript + Reagent instead of vanilla JS. All state in Reagent atoms.
- Supplementary docs: rationale, stats, compare, and learning-guide pages added
  as hash routes alongside the trainer.

* Routes
| Hash fragment    | Page component               | Description                      |
|------------------+------------------------------+----------------------------------|
| #/ or empty      | ``views/trainer-page``       | Typing trainer (primary feature) |
| #/layout         | ``layout/page``              | Enthium key map + stats          |
| #/learn          | ``views/learn-page``         | All documentation sections       |
| #/compare        | ``views/compare-page``       | Layout comparison table          |
| #/sections/TOPIC | ``views/section-page``       | Individual section               |

Routing is handled in ``router.cljs`` via a single Reagent atom ``current-route``.
``core.cljs`` dispatches on ``(:page route)`` with ``case``.

To add a new route:
1. Add the pattern + map to ``parse-hash`` in ``router.cljs``.
2. Add a ``navigate-to-X!`` helper in ``router.cljs``.
3. Add the ``case`` branch in ``core.cljs``.
4. Add the page component in ``views.cljs``.

* Typing Trainer Architecture
All trainer state lives in a single Reagent atom in ``trainer.cljs``.

** State shape
#+BEGIN_SRC clojure
(def state
  (r/atom
    {:level          1         ; 1–8
     :mode           :words    ; :words | :sentences | :time
     :limit          50        ; word count or seconds depending on mode
     :words          []        ; current prompt word list
     :word-index     0         ; index into :words of the active word
     :input          ""        ; current text-field value
     :hits           0         ; correctly completed words
     :errors         0         ; incorrect keystrokes
     :started?       false     ; true once first keystroke received
     :finished?      false     ; true when limit reached
     :start-ms       nil       ; js/Date.now() at first keystroke
     :elapsed-ms     0         ; updated by RAF loop while running
     :show-cheat?    true      ; show cheatsheet keyboard
     :sound-stroke?  false     ; play sound on keystroke
     :sound-error?   false     ; play sound on error
     :allow-caps?    false     ; include capital letters in prompts
     :allow-punct?   false     ; include punctuation in prompts
     :backspace-req? false}))  ; require backspace to correct errors
#+END_SRC

** Level definitions
Defined as a vector of maps in ``trainer.cljs``. Each level adds letters to the
unlocked set. Letter order follows Enthium v14 home-row frequency (highest first),
not Colemak order.

#+BEGIN_SRC clojure
(def levels
  [{:id 1 :label "Level 1" :letters #{"h" "t" "n" "s" "e" "a" "i" "r"}}
   {:id 2 :label "Level 2" :letters #{"o" "c" "l" "d"}}
   {:id 3 :label "Level 3" :letters #{"u" "y" "m" "p"}}
   {:id 4 :label "Level 4" :letters #{"b" "w" "f" "g"}}
   {:id 5 :label "Level 5" :letters #{"k" "v" "j" "x"}}
   {:id 6 :label "Level 6" :letters #{"'" "," "." ";"}}
   {:id 7 :label "Level 7" :letters nil}   ; all words
   {:id 8 :label "Level 8" :sentences? true}]) ; full sentences
#+END_SRC

Each level's available set is the union of all letter sets up to and including
that level. ``words.cljs`` filters the word list to only words constructable
from the current available-letters set.

** Trainer loop
1. ``init-round!`` — pick N words from the filtered list, reset indices/input.
2. User types in ``#userInput`` (a plain ``<input>``).
3. On each ``on-change``: compare current token against ``(nth words word-index)``.
   - Trailing space = word submitted. If correct → increment hits, advance index.
   - Wrong character → increment errors (optionally play error sound).
4. On word-limit or time-limit reached → set ``:finished? true``, compute WPM.
5. ``reset-round!`` resets state and calls ``init-round!``.

WPM formula (standard): ``(/ hits (/ elapsed-ms 60000))``.

** Cheatsheet keyboard
Rendered by ``layout/keyboard-diagram`` (already implemented). Shown below the
typing area when ``:show-cheat? true``. Toggled by a button in the toolbar.
Highlights the current target key based on the next character to type (future
enhancement — not required for v1).

* Documentation Sections
Secondary content. Each section lives in ``src/enthalpy/sections/s_TOPIC.cljs``.

** Section map shape
#+BEGIN_SRC clojure
(ns enthalpy.sections.s-learn)

(def section
  {:id       "learn"
   :featured true              ; show on learn-page featured row
   :title    "Learning Enthium v14"
   :subtitle "A structured onramp for switching from QWERTY or Colemak"
   :excerpt  "2-3 sentence blurb shown on section list card."
   :content
   [[:p "Opening paragraph..."]
    [:h2 "Section Header"]
    [:p "Body text."]]})
#+END_SRC

** Adding a new section
1. Create ``src/enthalpy/sections/s_TOPIC.cljs`` following the shape above.
2. In ``src/enthalpy/data.cljs``:
   - Add require: ``[enthalpy.sections.s-TOPIC :as s-TOPIC]``
   - Add ``s-TOPIC/section`` to the registry vector.
3. Run the syntax check — all files must print OK.
4. Run ``bun run dev`` and verify the section appears at ``#/learn``.

** Section ordering
``data/all-sections`` sorts by ``:id`` alphabetically.
Registry order does not matter.

* Hiccup Authoring Reference
The ``:content`` value is a vector of Reagent hiccup vectors rendered inside
``<article class="section-body">``.

** Available elements
| Hiccup                              | Renders as                        |
|-------------------------------------+-----------------------------------|
| ``[:p "..."]``                      | Body paragraph                    |
| ``[:h2 "..."]``                     | Section header                    |
| ``[:h3 "..."]``                     | Subsection header                 |
| ``[:ul [:li "..."] ...]``           | Bulleted list                     |
| ``[:ol [:li "..."] ...]``           | Numbered list                     |
| ``[:code "..."]``                   | Inline monospace (key names)      |
| ``[:strong "..."]``                 | Bold                              |
| ``[:em "..."]``                     | Italic                            |
| ``[:div.highlight-box [:p ...]]``   | Left-border insight callout       |
| ``[:div.key-box [:p ...]]``         | Left-border key sequence callout  |

** Special characters — never use HTML entities
| Character | ClojureScript literal |
|-----------+-----------------------|
| em dash   | ``\u2014``            |
| middot    | ``\u00B7``            |
| ×         | ``\u00D7``            |
| copyright | ``\u00A9``            |
| "         | ``\"`` (escaped)      |

** Styling rules
- Never add ``style=`` attributes to hiccup. Use CSS classes only.
- Never create per-section or per-page CSS overrides.
- Never hand-edit ``docs/js/``.
- New components: add CSS class to ``docs/css/style.css`` and add a mobile
  breakpoint under ``@media (max-width: 768px)``.

#+BEGIN_SRC sh
clj -M -e "(doseq [f (file-seq (clojure.java.io/file \"src\"))] (when (.endsWith (str f) \".cljs\") (try (read-string (str \"(do \" (slurp f) \")\")) (println \"OK:\" f) (catch Exception e (println \"ERR:\" f \"-\" (.getMessage e))))))"

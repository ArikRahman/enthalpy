* Overview

This document is the canonical style reference for all projects under the =arik.zip= umbrella.
Any new site, tool, or document that should feel visually consistent with the main blog should
treat this file as ground truth.

The aesthetic is: *dark, editorial, minimal, warm gold*. Inspired by high-end developer portfolios
and print typography. Quiet by default — accent colour only appears when it earns it.

-----

* Design Tokens

All values below should be expressed as CSS custom properties on =:root=.

#+begin_src css
:root {
    --bg:          #1c1c1e;   /* page / canvas background         */
    --bg-2:        #25262b;   /* hover state, secondary bg        */
    --bg-card:     #2c2e33;   /* card surfaces, code blocks       */
    --fg:          #f0ebe3;   /* primary text (warm off-white)    */
    --fg-muted:    #a6a7ab;   /* body copy, secondary text        */
    --fg-subtle:   #5c5f66;   /* de-emphasised, placeholders      */
    --accent:      #c9a96e;   /* gold — the only colour that pops */
    --border:      #373a40;   /* default dividers / outlines      */
    --border-mid:  #414550;   /* stronger borders, code outlines  */
}
#+end_src

** Colour Usage Rules

| Token          | Use for                                              | Never use for                     |
|----------------+------------------------------------------------------+-----------------------------------|
| =--accent=     | CTAs, hover states, active labels, links, markers    | Large fills, backgrounds          |
| =--fg=         | Headings, strong emphasis, logo                      | Long body copy                    |
| =--fg-muted=   | Body paragraphs, card descriptions, meta text        | Primary headings                  |
| =--fg-subtle=  | Post numbers, dates, list markers, separators        | Anything interactive              |
| =--bg-card=    | Card backgrounds, inline code chips, empty states    | Full-page backgrounds             |
| =--border=     | Section dividers, card outlines at rest              | Anything that needs contrast      |
| =--border-mid= | Stronger outlines (code, avatar, hover borders)      | Default rest state                |

** Accent Tints

Used for glow/wash effects — never as solid fills.

#+begin_src css
rgba(201, 169, 110, 0.06)   /* highlight-box background wash    */
rgba(201, 169, 110, 0.09)   /* hero radial glow                 */
rgba(201, 169, 110, 0.045)  /* body background radial gradient  */
rgba(201, 169, 110, 0.25)   /* ::selection highlight            */
rgba(201, 169, 110, 0.28)   /* blog link underline at rest      */
rgba(201, 169, 110, 0.35)   /* gradient fade tail on card top   */
#+end_src

** Gradient Text Pattern

Used for all major headings (hero name, page titles, post titles).

#+begin_src css
background: linear-gradient(150deg, #f5efe6 0%, var(--fg) 45%, #c9a050 100%);
-webkit-background-clip: text;
-webkit-text-fill-color: transparent;
background-clip: text;
#+end_src

The gradient goes from near-white → warm cream → gold. The angle (150deg) creates a
subtle diagonal sweep. Do not use for anything below =h1= level.

| List items       | =1.75–1.8=  |

* Layout

** Container

#+begin_src css
.container {
    max-width: 900px;
    margin: 0 auto;
    padding: 0 40px;
}
#+end_src

Most content sections implement this constraint inline (=max-width: 900px; margin: 0 auto=)
rather than wrapping in a shared =.container= div. The =.container= class is reserved for
inner page layouts (writing page, post page, CV page).

** Body Background

The page background gets a subtle warm radial wash at the very top:

#+begin_src css
background: var(--bg);
background-image: radial-gradient(
    ellipse 100% 45% at 50% 0%,
    rgba(201, 169, 110, 0.045) 0%,
    transparent 60%
);
#+end_src

** Grid Patterns

Cards use auto-fill responsive grid:

#+begin_src css
grid-template-columns: repeat(auto-fill, minmax(260px, 1fr));
gap: 16px;
#+end_src

Post items use a 3-column grid:

#+begin_src css
grid-template-columns: 52px 1fr auto;  /* number | body | meta */
align-items: center;
gap: 0 28px;
#+end_src

** Avatar

#+begin_src css

    width: 160px; height: 160px;

** Layout

Desktop: =flex-direction: row=, =gap: 64px=, avatar left / text right.
Mobile: =flex-direction: column=, =align-items: center=, =text-align: center=.

** Eyebrow → Name → Subtitle → Bio → CTA

The visual hierarchy of the hero follows a strict reading order:
1. Eyebrow label (=0.75rem=, uppercase, =--accent=, =margin-bottom: 24px=)
2. Name (gradient heading, =margin-bottom: 36px=)
3. Animated role subtitle (=0.75rem= em dash prefix + fading role text, =margin-bottom: 32px=)
4. Bio paragraph (=0.9375rem=, =--fg-muted=, =max-width: 460px=, =margin-bottom: 52px=)
5. CTA buttons (=gap: 14px=)

** Animated Subtitle

The animated role text fades out (=opacity: 0=, =translateY(10px)=), swaps text, then fades back in.
Transition: =opacity 0.35s ease= + =transform 0.35s ease=. Interval: =3200ms=. Fade duration: =380ms=.

* Motion & Transitions

The site uses exclusively =ease= timing with short durations. There are no bouncy, spring, or cubic-bezier curves — everything is smooth and restrained.

| Interaction              | Property              | Duration | Easing |
|--------------------------+-----------------------+----------+--------|
| All colour changes       | =color=, =background= | =0.2s=   | =ease= |
| Nav link underline       | =width=               | =0.25s=  | =ease= |
| Card hover lift          | =transform=, =shadow= | =0.2s=   | =ease= |
| Card top bar sweep       | =transform: scaleX=   | =0.3s=   | =ease= |
| Post item indent         | =padding-left=        | =0.25s=  | =ease= |
| Arrow nudge (→)          | =transform: translateX= | =0.2s= | =ease= |
| Role text fade           | =opacity=, =transform= | =0.35s= | =ease= |
| CTA / button             | =all=                 | =0.2s=   | =ease= |

**Rule**: Never use =all= outside of a button context. Be explicit about which properties transition. No =transition-delay=.

* Responsive Breakpoints

Two breakpoints only.

** 768px — Tablet / Mobile

#+begin_src
- Nav: padding shrinks to 24px sides
- Nav links: gap shrinks from 40px → 24px
- Hero: column layout, centre-aligned text
- Stack grid: single column
- Post item: 2-column grid (number + body), meta row below
- Post item arrow: hidden
- Container padding: 40px → 24px
- Footer: column layout
- CV entry header: stacks vertically
#+end_src

** 480px — Small Mobile

#+begin_src
- Nav: last link hidden (Contact)
- Hero name: letter-spacing loosens slightly
- CTA buttons: full width column stack
- Post item: padding 28px 0
#+end_src

* Icons

SVG icons are inline components. Size is passed as a prop (default =18px=).
Colour is inherited from the parent (=currentColor=) so hover states propagate automatically.
Never use icon fonts or image-based icons.

-----

* Do / Don't Quick Reference

| ✅ Do                                              | ❌ Don't                                          |
|---------------------------------------------------+----------------------------------------------------|
| Use CSS custom properties for every colour        | Hardcode hex values in component styles            |
| Use =--accent= sparingly, only to highlight       | Use =--accent= for large fills or backgrounds      |
| Animate only specific properties                  | Use =transition: all= (except buttons)             |
| Use Unicode characters directly                   | Use HTML entities (=&mdash;= etc.)                 |
| Add mobile breakpoints for every new component   | Add responsive styles inline via =style==          |
| Keep border-radius minimal (3px–6px)              | Use large rounded corners (>6px)                   |
| Use =clamp()= for fluid heading sizes             | Use fixed =px= sizes for major headings            |
| Use gradient text only on =h1= / page titles      | Apply gradient text to body copy or small text     |
| Keep the motion quiet (0.2–0.3s ease)             | Add bounce, spring, or delay-heavy animations      |
| Write CSS classes; add to =style.css=             | Write inline =style== attributes                   |

-----

* Checklist for New Projects

When bootstrapping a new project under this design system:

- [ ] Import Inter from Google Fonts (weights 300–900)
- [ ] Define all =:root= CSS custom properties
- [ ] Set body background with =radial-gradient= warm wash
- [ ] Enable =-webkit-font-smoothing: antialiased=
- [ ] Style =::selection= with accent tint
- [ ] Style =::-webkit-scrollbar= to match dark theme
- [ ] Reset =*, *::before, *::after= (margin, padding, box-sizing)
- [ ] Set =scroll-behavior: smooth= on =html=
- [ ] Set =color: inherit= and =text-decoration: none= on =a=
- [ ] Implement fixed frosted-glass nav
- [ ] Use =max-width: 900px= container
- [ ] Add =@media (max-width: 768px)= and =@media (max-width: 480px)= blocks
